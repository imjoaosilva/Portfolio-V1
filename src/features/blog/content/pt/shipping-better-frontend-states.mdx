A forma mais fácil de fazer uma interface polida parecer inacabada é desenhar só o happy path. As pessoas percebem primeiro as bordas: o spinner que nunca muda, a mensagem de erro que não ajuda e o estado vazio que parece acidental.

## Trate o assíncrono como máquina de estados

Eu gosto de tratar toda fronteira assíncrona como uma pequena máquina de estados. O que a pessoa vê antes dos dados chegarem? O que acontece quando não existe dado? E quando só parte dele chegou? Quando esses estados ficam explícitos, a estrutura do componente costuma ficar mais simples, não mais complexa.

## Divida responsabilidade com honestidade

Em apps com Next, muita clareza vem de separar bem as responsabilidades. Server components podem buscar e moldar os dados. Partes menores e visuais podem focar em renderizar cada estado sem precisar controlar o fluxo inteiro.

O resultado não é só uma UX melhor. Também reduz condicionais espalhadas pela árvore. O componente começa a parecer uma especificação de produto, não uma sequência de remendos defensivos.
