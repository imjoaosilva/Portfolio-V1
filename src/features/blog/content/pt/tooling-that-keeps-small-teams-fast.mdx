Times rápidos raramente são rápidos por causa de uma ferramenta brilhante. Eles são rápidos porque o caminho básico está bem pavimentado. Instalar funciona. O lint é previsível. Os nomes são consistentes. Os PRs são fáceis de escanear. O básico foi mantido com intenção.

## Faça o caminho padrão ser suave

Quando o projeto ainda é pequeno, é tentador adiar essas decisões. Mas é exatamente nessa fase que elas custam menos. Um pouco de estrutura cedo economiza uma quantidade surpreendente de tempo quando mais pessoas começam a tocar as mesmas áreas.

## Automatize só o que protege foco

Minha preferência é automatizar apenas o que preserva foco. Um script que remove setup repetitivo é ótimo. Um pipeline complexo que ninguém entende geralmente não é. O melhor tooling parece guardrail, não labirinto.

Se o fluxo está limpo, dá para gastar energia em problema de produto em vez de reaprender os rituais do repositório toda semana.
